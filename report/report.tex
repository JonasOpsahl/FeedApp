\title{DAT250 FeedApp Report}

\author{Erlend André Høntorp, Jonas Heen Opsahl and Thomas Tolo Jensen}
\onehalfspacing

\maketitle

\begin{abstract}
Live polling applications need to keep data consistent across many users 
while staying fast and responsive. FeedApp is a real-time polling platform 
where users can create polls, vote, and discuss results in comment threads, 
with updates appearing instantly for everyone. This project evaluates Apache 
Kafka against RabbitMQ as event streaming platforms for handling real-time 
updates in web applications. We built the system with a Spring Boot backend 
and React frontend, using Kafka's topic-per-poll approach along with Redis 
for caching and WebSockets for push notifications. Through benchmarking with 
OpenMessaging Benchmark framework, we tested hypotheses around throughput, 
scalability, and latency under different workloads simulating high-volume 
vote ingestion and real-time fan-out to clients. Our experiments showed that 
Kafka achieved significantly higher throughput and better horizontal scaling 
for write-heavy workloads, while RabbitMQ demonstrated lower latency at low 
message rates due to its push model. However, Kafka's performance advantage 
increased under high load as RabbitMQ's per-message overhead became more 
costly. Creating separate Kafka topics for each poll worked well for keeping 
things isolated, though it required extra management overhead. The main 
takeaways from this project are understanding the throughput-latency trade-offs 
between log-based and queue-based messaging systems, when event-driven 
architectures like Kafka justify their complexity, and how to handle cache 
invalidation properly in distributed systems.
\end{abstract}
